This repository contains source code, scripts, and configuration files developed for Cisco DevNet Associate (DEVASC) labs and network automation workflows. It focuses on programmatically managing network infrastructure using Python, REST APIs, and modern automation tools.

📁 𝗥𝗲𝗽𝗼𝘀𝗶𝘁𝗼𝗿𝘆 𝗦𝘁𝗿𝘂𝗰𝘁𝘂𝗿𝗲

WEBEX-TEAMS: Scripts and integration code for interacting with the Webex Teams (now Webex App) API, likely for notifications or chatbot automation.

ansible: Playbooks and inventory files focused on network automation, including IPv6 configurations and verification for Cisco CSR routers.

graphhopper: Implementations or configurations for the GraphHopper routing engine, exploring different modes of transportation and pathfinding.

jenkins: CI/CD pipeline configurations and scripts, including logic for password evolution and automated build processes.

netconf: Integrated lab scripts for automating Cisco Catalyst 8000v (Cat8kv) devices using the NETCONF protocol.

packet tracer: Lab files and documentation for implementing REST APIs with SDN controllers within the Cisco Packet Tracer environment.

parsing: Code focused on data serialization and structural conversion, specifically dealing with YAML and other data formats.

ptna: Additional resources for Packet Tracer Network Automation, specifically regarding SDN controller API implementations.

pyang: Files related to YANG model validation and conversion using the pyang toolset.

pyats/csr1kv: Automated testing scripts using the Cisco pyATS framework specifically tailored for the CSR1000v platform.

python: Core Python programming exercises and modules focusing on object-oriented principles like classes and methods.

restconf: Scripts for managing network resources via the RESTCONF protocol, such as configuring Loopback interfaces using PUT requests.

sample-app: A demonstration application, including Dockerfiles for containerization and deployment testing.

📝 𝗡𝗼𝘁𝗲𝘀

Most scripts are designed to be run against Cisco DevNet Always-On or Reserved sandboxes.

Ensure your Ubuntu environment has net-tools and openssh-client installed for full functionality.

security: Resources and scripts dedicated to network security updates and hardening configurations.

unittest: Python unit tests designed to validate code logic and perform bug fixes across the repository's modules.
